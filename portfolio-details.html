Web Scraper

  • Language: Python
  • New Concepts: HTTP Requests, Dictionaries, BeautifulSoup, OS Module, File Handling
  • Github URL:

Short write up of the project

The scraper works by sending a request to the URL, in this case nature.com, and asking the user to enter the category of article they would like to parse and the number of pages to look for these articles in. Using the Beautiful Soup library to find the articles by category then using the OS module to create a new directory for each of the articles parsed and save the contents in a txt file for reading offline. The script can be repurposed to be used for a number of websites and is faster than using a web browser to manually search through each page of a news site to get articles.

The project required me to read the python documentation to get a good understanding of the concepts needed to complete the project. The way that the tests are run gives a lot of room for creativity as there are many ways to get the program to work as it should. Despite this creative license, the fundamental concepts of using Requests, BeautifulSoup, the OS module, dictionaries and handling files in a script, had to be used in the most efficient manner to create a working web scraper. A lot of scratch files were created in order to check if each block of code did what it was supposed to. The project was challenging because it covered a lot of topics at once but this allowed me to explore a lot of new ways of writing Python and in the end I was able to create a program that communicates with the internet.
